How the GA works
1
Chromosome (Phase 1) — Each individual is an (n_classes × 2) int array. Column 0 = chosen room index, Column 1 = chosen time index, both into each class's own candidate list from the XML.
2
Fitness — fitness = hard_weight × hard_violations + soft_penalties. Hard violations (room conflicts, instructor double-booking, room sharing) are penalised ×1000 to make them effectively non-negotiable.
3
Selection — Tournament selection (k=3). Each round picks 3 random individuals and keeps the one with the lowest fitness.
4
Crossover — Uniform crossover: for each class (row), independently inherit from parent A or B with 50/50 probability.
5
Chromosome (Phase 2) — A permutation of student IDs. The order in which students are greedily assigned to sections determines who gets their preferred section first.
6
Phase 2 Crossover — Order Crossover (OX): copies a contiguous segment from parent A, fills the rest from parent B in order. Required for valid permutations.
