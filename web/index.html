How the GA works
1
Chromosome (Phase 1) — Each individual is an (n_classes × 2) int array. Column 0 = chosen room index, Column 1 = chosen time index, both into each class's own candidate list from the XML.
2
Fitness — fitness = hard_weight × hard_violations + soft_penalties. Hard violations (room conflicts, instructor double-booking, room sharing) are penalised ×1000 to make them effectively non-negotiable.
3
Selection — Tournament selection (k=3). Each round picks 3 random individuals and keeps the one with the lowest fitness.
4
Crossover — Uniform crossover: for each class (row), independently inherit from parent A or B with 50/50 probability.
5
Chromosome (Phase 2) — A permutation of student IDs. The order in which students are greedily assigned to sections determines who gets their preferred section first.
6
Phase 2 Crossover — Order Crossover (OX): copies a contiguous segment from parent A, fills the rest from parent B in order. Required for valid permutations.
Asymptotic Complexity
Naive Search Space
O( (|R|·|T|)n )
n = classes  ·  |R| = rooms  ·  |T| = timeslots
!
Exponential — naïve search: Each class picks one of |R|×|T| room-time pairs independently. Total combinations = (|R|×|T|)^n. With 63 rooms × 288 time cells = 18,144 options/class → 18,144^896 ≈ 103,559 states. Completely intractable for exact search.
↓
Candidate filtering (pruning): Pre-filter each class to rooms that fit capacity/lab type and times where the instructor is available. Reduces average domain from 18,144 → ~100 options/class, shrinking the search space by many orders of magnitude — but still exponential.
✓
Genetic Algorithm — polynomial in practice: Only evaluates P×G fitness calls where P = population size and G = generations. Each evaluation is O(n). Total work ≈ O(P·G·n) — polynomial, not exponential. GA does not guarantee the global optimum but reliably finds high-quality feasible solutions.

1
Partition: The full problem is split into two independent sub-problems — room/time assignment and student sectioning — each with its own chromosome, fitness function, and search operators.
2
Conquer (Phase 1): Runs a GA on the (room, time) sub-problem. Fitness = hard_weight × hard_violations + soft_penalties. Result: a fixed room/time assignment for every class.
3
Conquer (Phase 2): Takes the Phase 1 schedule as a fixed constraint. Runs a separate GA over student processing order using Order Crossover (OX) to preserve valid permutations.
4
Merge: Phase 2 greedily assigns each student to an available section in the Phase 1 timetable, respecting time conflicts and capacity limits. The two solutions combine into the final schedule.
5
Why it works: Decoupling reduces the joint search space from O((|R|·|T|·|S|)^n) to two tractable sub-problems, allowing each GA to converge faster on a focused objective.

University Course Timetabling Problem

Genetic Algorithm Two-Phase Solver 10,000+ Students 500+ Courses MIT License
Overview

UniversityTimeTable is a two-phase Genetic Algorithm (GA) solver for the University Course Timetabling Problem (UCTP), using a real dataset from Purdue University (Fall 2007). The system produces a conflict-free schedule with minimized idle gaps across 10,000+ students and 500+ courses.

Phase 1
Course Scheduling
Assign 896+ course sections to rooms and time slots using a GA with uniform crossover and hard/soft constraint fitness.
896 classes · 63 rooms · 2016 time cells
Phase 2
Student Sectioning
Assign 30,000+ students into scheduled sections using a GA over student-processing order with greedy placement.
30,846 students · Order Crossover (OX)
Goals
🎯
Global feasibility — Generate a conflict-free schedule for a simulated university of 10,000+ students.
📐
Optimization — Maximize compactness by minimizing idle gaps between lectures for student cohorts.
📊
Scalability & clarity — Show runtime & improvement curves, explain ordering/pruning, provide one-command reproducibility with fixed seed.
📦
Professional deliverable — Organized code, concise README, useful outputs and plots.
Constraints
🔴 Hard — must be zero
  • Non-Overlap — No double-booking of rooms, instructors, or cohorts
  • Room capacity — Enrollment must not exceed available seats
  • Resource & suitability — Room must have required equipment
  • Course structure — Valid lecture + lab combinations maintained
  • Instructor qualification — Only qualified instructors assigned
  • Instructor availability — Blocked times are non-negotiable
  • Travel time limits — Back-to-back classes allow feasible walk time
🟡 Soft — minimize penalty
  • Student gap minimization — Reduce idle dead hours within a day
  • Hybrid modeling — Allow hybrid/online courses to use a virtual room
  • Workload distribution — Reasonable daily/consecutive teaching limits
  • Preference satisfaction — Honor non-required faculty time preferences
